## 核心概念（读完前两篇就能理解）

| 概念 | 一句话 |
|------|--------|
| **Module** | 像个收纳盒，把 Controller 和 Service 组织在一起 |
| **Controller** | 定义路由，提取参数，不写业务逻辑 |
| **Service** | 写业务逻辑和数据操作的地方 |
| **DTO** | 定义"前端发来的数据长什么样" |
| **Pipe** | 管道，数据来了先"洗一遍"（校验/转换） |
| **Guard** | 守卫，决定"这个请求能不能进来" |
| **Interceptor** | 拦截器，在请求前后"插一脚"（日志/转换） |
| **Filter** | 过滤器，异常发生时"统一收拾残局" |
| **Decorator** | 自定义装饰器，给你的参数/方法贴标签 |
