# ⑥ 命令速查

## 开发启动

| 命令 | 用途 |
|------|------|
| `pnpm start:dev` | **最常用** — 开发模式，文件修改自动重启 |
| `pnpm start` | 启动一次，不监听文件变化 |
| `pnpm start:debug` | 调试模式，可附加 Node.js inspector |
| `pnpm start:prod` | 生产模式（需先 `pnpm build`） |

## 构建

| 命令 | 用途 |
|------|------|
| `pnpm build` | 编译 TypeScript 到 `dist/` 目录 |

## 数据库

| 命令 | 用途 |
|------|------|
| `pnpm prisma:generate` | 生成 Prisma Client 类型（首次或改 schema 后必执行） |
| `pnpm prisma:push` | 同步 `schema.prisma` 到数据库 |
| `pnpm prisma:studio` | 打开浏览器数据库管理界面 |

## 代码质量

| 命令 | 用途 |
|------|------|
| `pnpm lint` | ESLint 代码检查 + 自动修复 |
| `pnpm format` | Prettier 代码格式化 |

## 测试

| 命令 | 用途 |
|------|------|
| `pnpm test` | 运行单元测试 |
| `pnpm test:watch` | 监听模式自动重跑测试 |
| `pnpm test:cov` | 测试 + 覆盖率报告 |
| `pnpm test:e2e` | 端到端测试 |

## 依赖管理

| 命令 | 用途 |
|------|------|
| `pnpm add <pkg>` | 安装生产依赖 |
| `pnpm add -D <pkg>` | 安装开发依赖 |
| `pnpm update --latest` | 升级所有依赖到最新版本 |
| `pnpm ls --depth=0` | 查看直接依赖列表 |

## 实用工具

```bash
# 查看当前运行的 NestJS 路由列表
pnpm start:dev  # 启动后在日志中可以看到所有 Mapped 路由

# 查看数据库数据
pnpm prisma:studio

# 查看 Swagger API 文档（启动服务器后）
open http://localhost:3000/api
```
