# ① 快速启动

## 前置条件

- Node.js >= 20（当前版本 22.17.1）
- pnpm（包管理器，当前版本 11.5.2）

## 安装依赖

```bash
# 在根目录执行一次，自动安装 server/ + web/ 全部依赖
pnpm install
```

## 初始化数据库

项目使用 Neon PostgreSQL（已配置好远程数据库），只需生成 Prisma Client 类型：

```bash
pnpm --filter @nestjs-demo/server prisma:generate
```

## 启动开发服务器

### 启动后端（端口 3000）

```bash
pnpm --filter @nestjs-demo/server start:dev
```

### 启动前端（端口 5173，新开终端）

```bash
pnpm --filter web dev
```

终端看到以下日志表示启动成功：

```
[NestApplication] Nest application successfully started
```

前端页面访问 http://localhost:5173，API 请求自动代理到后端 3000 端口。

## 测试接口

项目已预置测试数据（2 个用户 + 2 篇文章）。用 curl 或浏览器测试：

```bash
# 用户列表
curl http://localhost:3000/users

# 用户详情（含文章）
curl http://localhost:3000/users/1

# 文章列表（含作者）
curl http://localhost:3000/posts

# Swagger API 文档
open http://localhost:3000/api
```

## 完整测试示例

```bash
# 注册
curl -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@test.com","password":"123456"}'

# 登录
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@test.com","password":"123456"}'

# 创建文章（需 Token）
curl -X POST http://localhost:3000/posts \
  -H "Content-Type: application/json" \
  -H 'Authorization: Bearer <Token>' \
  -d '{"title":"Hello","content":"World","authorId":1}'

# 删除
curl -X DELETE http://localhost:3000/users/1 \
  -H 'Authorization: Bearer <Token>'
```

## 常见问题

**Q: 端口 3000 被占用？**
```bash
lsof -ti:3000 | xargs kill -9
```

**Q: Prisma Client 类型找不到？**
```bash
pnpm --filter @nestjs-demo/server prisma:generate
```
