# NestJS Demo — 入门学习文档

一个基于 NestJS 11 + Prisma 7 + Neon PostgreSQL + React 19 的**前后端分离 monorepo 项目**，适合学习 NestJS
的模块化架构和分层设计。**覆盖 18 个 NestJS 核心概念**，每个都有真实 API 端点和可调用接口。

## 项目架构

```
nestjs-demo/                    # pnpm monorepo 根目录
├── package.json                # workspace 管理
├── pnpm-workspace.yaml         # 定义子包
├── README.md                   # monorepo 使用说明
│
├── server/                     # NestJS 后端（本文档目录）
│   ├── src/                    #   44 个 .ts 源文件
│   ├── docs/                   #   本文档
│   └── ...
│
└── web/                        # React 19 + Ant Design 6 前端
    ├── src/
    │   ├── api/                #   Axios API 封装（6 个模块）
    │   ├── stores/             #   Zustand 状态管理
    │   ├── pages/              #   8 个页面
    │   └── ...
    └── ...
```

## 核心概念（18 个，读完前两篇就能理解）

| # | 概念 | 一句话 |
|---|------|--------|
| 1 | **控制器** | 定义路由，提取参数，不写业务逻辑 |
| 2 | **提供者** | 封装可复用逻辑，依赖注入自动管理 |
| 3 | **模块** | 像个收纳盒，把 Controller 和 Service 组织在一起 |
| 4 | **中间件** | 请求拦截器，每个请求都过一遍（如计时） |
| 5 | **异常过滤器** | 异常发生时"统一收拾残局" |
| 6 | **管道** | 数据来了先"洗一遍"（校验/类型转换） |
| 7 | **守卫** | 决定"这个请求能不能进来" |
| 8 | **拦截器** | 在请求前后"插一脚"（日志/转换） |
| 9 | **自定义装饰器** | 给你的方法/参数贴标签 |
| 10 | **认证** | 你是谁？注册/登录验证 |
| 11 | **授权** | 你能干什么？角色权限控制 |
| 12 | **JWT** | 签发和验证 Token，无状态认证 |
| 13 | **自定义提供者** | useFactory 运行时动态提供值 |
| 14 | **异步提供者** | async useFactory 启动时检查数据库 |
| 15 | **动态模块** | forRoot 模式，运行时传参配置模块 |
| 16 | **生命周期事件** | 模块初始化/关闭时自动触发 |
| 17 | **执行上下文** | Guard/Interceptor 拿到当前路由信息 |
| 18 | **文件上传** | Multer 封装，两个装饰器搞定上传 |
