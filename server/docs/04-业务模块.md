# ④ 业务模块（modules/）

业务模块遵循一样的四层架构模式：**Module → Controller → Service → DTO**。

## 模块通用模板

```
modules/xxx/
├── xxx.module.ts         ← @Module({ controllers, providers })
├── xxx.controller.ts     ← @Controller('xxx') 路由定义
├── xxx.service.ts        ← @Injectable() 业务逻辑
├── dto/
│   ├── create-xxx.dto.ts ← 创建请求体定义 + 校验规则
│   └── update-xxx.dto.ts ← PartialType(CreateXxxDto) 所有字段可选
└── entities/             ← 占位（Prisma 不需要实体文件）
```

## Users vs Posts 对比

两个模块结构完全一致，通过对比可以看出"哪些是模板代码，哪些是业务差异"。

| 方面 | UsersModule | PostsModule |
|------|-------------|-------------|
| **Module** | 注册 `UsersController + UsersService` | 注册 `PostsController + PostsService` |
| **Controller** | `@Controller('users')` | `@Controller('posts')` |
| **Service 注入** | `db: DatabaseService` | `db: DatabaseService` |
| **Prisma 操作** | `db.user.create/findMany/findUnique/update/delete` | `db.post.create/findMany/findUnique/update/delete` |
| **关联查询** | `include: { posts: true }` | `include: { author: true }` |
| **创建校验** | 校验 author 是否存在 | 校验 author 是否存在 |
| **异常处理** | P2002（唯一冲突）+ P2025（不存在） | P2025（不存在） |
| **DTO** | CreateUserDto, UpdateUserDto | CreatePostDto, UpdatePostDto |

## DTO 设计差异

**CreateUserDto** — 所有字段必填：
```typescript
@IsString() @IsNotEmpty()  username!: string;
@IsEmail()                  email!: string;
```

**CreatePostDto** — 有可选字段 + 数值校验：
```typescript
@IsString() @IsNotEmpty()    title!: string;
@IsString() @IsOptional()    content?: string;
@IsInt() @IsPositive()        authorId!: number;
```

## PartialType 模式

两个模块的 Update DTO 使用完全相同的模式：
```typescript
export class UpdateXxxDto extends PartialType(CreateXxxDto) {}
```

`PartialType` 把 Create 中的所有字段变为可选，正好满足"更新时只需传要改的字段"的需求。NestJS 还提供了 `PickType`、`OmitType`、`IntersectionType` 等工具，用于组合 DTO。

## 接口清单

| 方法 | 路由 | Users 响应示例 | Posts 响应示例 |
|------|------|---------------|---------------|
| POST | `/{resource}` | 201 + 用户 JSON | 201 + 文章 JSON（含 author） |
| GET | `/{resource}?page=1&limit=10` | 用户数组 + 每人文章 | 文章数组 + 每人作者 |
| GET | `/{resource}/:id` | 单个用户 + 文章列表 | 单篇文章 + 作者信息 |
| PATCH | `/{resource}/:id` | 更新后的用户 | 更新后的文章（含 author） |
| DELETE | `/{resource}/:id` | 204 No Content | 204 No Content |

所有列表接口支持分页：`GET /users?page=1&limit=20`
