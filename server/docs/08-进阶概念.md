# ⑧ 进阶概念 — 生命周期/动态模块/执行上下文/文件上传/自定义提供者

## 1. 生命周期事件（Lifecycle Events）

NestJS 在应用启动/关闭时会自动调用特定方法。类似于 React 的 `useEffect(() => {}, [])` 和 Vue 的 `onMounted` / `onUnmounted`。

```typescript
@Injectable()
export class LifecycleDemoService
  implements OnModuleInit, OnApplicationShutdown
{
  onModuleInit() {
    // 模块依赖解析完成后自动调用
    // 适用场景：连接数据库、加载配置、预热缓存
    console.log('【生命周期】模块初始化完成');
  }

  onApplicationShutdown(signal: string) {
    // 应用关闭时自动调用
    // 适用场景：关闭连接、释放资源
    // 需要在 main.ts 中 app.enableShutdownHooks() 才能生效
    console.log(`【生命周期】应用关闭，信号: ${signal}`);
  }
}
```

**执行顺序**：
```
constructor → onModuleInit → [应用运行中] → onApplicationShutdown → onModuleDestroy
```

**源文件**：`src/modules/demo/demo-lifecycle.service.ts`，已注册到 `app.module.ts`，启动日志可见。

## 2. 动态模块（Dynamic Module / forRoot）

动态模块是 NestJS 最强大的模式——几乎所有官方模块都用它。

**前端类比**：`Vue.use(Router, { mode: 'history' })`

**后端实例**：
- `TypeOrmModule.forRoot({ ... })` — TypeORM 配置
- `JwtModule.registerAsync({ ... })` — JWT 配置
- `ConfigModule.forRoot({ isGlobal: true })` — 配置模块

本项目的 `DynamicConfigModule`：

```typescript
@Module({})  // ← 空白的！内容在 forRoot 中动态返回
export class DynamicConfigModule {
  static forRoot(options: { path: string }): DynamicModule {
    return {
      module: DynamicConfigModule,
      global: true,
      controllers: [DynamicConfigController],
      providers: [
        { provide: 'CONFIG_PATH', useValue: options.path },  // 动态注入配置值
        DynamicConfigService,
      ],
      exports: [DynamicConfigService],
    };
  }
}
```

**关键区别**：普通模块的配置是写死的，动态模块的配置由使用方在 `imports: [XXX.forRoot({...})]` 时传入。

**验证方式**：`GET /dynamic-config` → 返回 `{ configPath: ".env", message: "由 forRoot 动态传入" }`

## 3. 执行上下文（ExecutionContext）

`ExecutionContext` 是 Guard/Interceptor/Filter 的核心参数，它知道"当前是哪个 Controller 的哪个方法被调用"。

```typescript
canActivate(context: ExecutionContext): boolean {
  // ExecutionContext 可以拿到：
  const handler = context.getHandler();           // 当前调用的 Controller 方法
  const controller = context.getClass();          // 当前调用的 Controller 类
  const request = context.switchToHttp().getRequest();  // HTTP 请求对象

  // 在 WebSocket/微服务场景下，同一个 Guard 也能用：
  // context.switchToWs()  → WebSocket 上下文
  // context.switchToRpc() → 微服务 RPC 上下文
}
```

**业务场景**：`RolesGuard` 通过 `context.getHandler()` 拿到当前方法，用 `Reflector` 读取该方法上的 `@Roles('admin')` 装饰器设置的元数据，判断当前用户是否有权限。

## 4. 文件上传（File Upload）

NestJS 用装饰器封装了 Multer，加两个装饰器就能实现文件上传。

```typescript
@Controller('files')
export class FilesController {
  @Post('upload')
  @UseInterceptors(FileInterceptor('file'))    // 'file' 是 form-data 中的字段名
  uploadFile(
    @UploadedFile() file: Express.Multer.File,  // 自动提取文件对象
  ) {
    // file 包含:
    //   originalname — 原始文件名, e.g. "photo.jpg"
    //   mimetype     — 文件类型, e.g. "image/jpeg"
    //   size         — 文件大小（字节）
    //   buffer       — 文件内容（二进制）
    return {
      originalname: file.originalname,
      mimetype: file.mimetype,
      size: file.size,
    };
  }
}
```

**前端调用方式**：
```javascript
const formData = new FormData()
formData.append('file', selectedFile)  // 字段名 'file' 必须和后端一致
axios.post('/files/upload', formData, {
  headers: { 'Content-Type': 'multipart/form-data' }
})
```

## 5. 自定义提供者（useFactory）

DI 容器不仅可以用 `@Injectable()` 自动注册，也可以手动用工厂函数创建。

```typescript
// app.module.ts providers 数组中
{
  provide: 'APP_CONFIG',  // token 名
  useFactory: (configService: ConfigService) => ({
    port: configService.get<number>('port', 3000),
    environment: process.env.NODE_ENV || 'development',
  }),
  inject: [ConfigService],  // 依赖的 provider
}
```

**类比**：Vue 的 `provide/inject`，只是 NestJS 可以在运行时根据其他 provider 动态计算值。

## 6. 异步提供者（async useFactory）

和 useFactory 一样，只是工厂函数是 async 的——适用于启动时需要异步操作。

```typescript
{
  provide: 'DATABASE_HEALTH',
  useFactory: async (db: DatabaseService) => {
    try {
      await db.$queryRaw`SELECT 1`;  // 执行 SQL 查数据库是否连通
      return { connected: true, timestamp: new Date() };
    } catch {
      return { connected: false, timestamp: new Date() };
    }
  },
  inject: [DatabaseService],
}
```

**特点**：模块会在异步操作完成后才完成加载，确保所有依赖在启动时就已经就绪。
