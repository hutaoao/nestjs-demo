# ⑥ 命令速查

> 由于项目已改为 pnpm monorepo 结构，所有命令需通过 `--filter @nestjs-demo/server` 指定后端包，
> 或在 `server/` 目录下直接执行。

## 开发启动

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server start:dev` | **最常用** — 开发模式，文件修改自动重启 |
| `pnpm --filter @nestjs-demo/server start` | 启动一次，不监听文件变化 |
| `pnpm --filter @nestjs-demo/server start:debug` | 调试模式，可附加 Node.js inspector |
| `pnpm --filter web dev` | 启动前端开发服务器（端口 5173） |
| `pnpm --filter web build` | 构建前端生产包 |

## 构建

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server build` | 编译 TypeScript 到 `server/dist/` 目录 |

## 数据库

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server prisma:generate` | 生成 Prisma Client 类型（首次或改 schema 后必执行） |
| `pnpm --filter @nestjs-demo/server prisma:push` | 同步 `schema.prisma` 到数据库 |
| `pnpm --filter @nestjs-demo/server prisma:studio` | 打开浏览器数据库管理界面 |

## 代码质量

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server lint` | ESLint 代码检查 + 自动修复 |
| `pnpm --filter @nestjs-demo/server format` | Prettier 代码格式化 |

## 测试

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server test` | 运行单元测试 |
| `pnpm --filter @nestjs-demo/server test:watch` | 监听模式自动重跑测试 |
| `pnpm --filter @nestjs-demo/server test:cov` | 测试 + 覆盖率报告 |
| `pnpm --filter @nestjs-demo/server test:e2e` | 端到端测试 |

## 依赖管理

| 命令 | 用途 |
|------|------|
| `pnpm --filter @nestjs-demo/server add <pkg>` | 安装后端生产依赖 |
| `pnpm --filter web add <pkg>` | 安装前端生产依赖 |
| `pnpm update --latest` | 升级所有依赖到最新版本 |
| `pnpm ls --depth=0` | 查看直接依赖列表 |

## API 测试命令（curl）

```bash
# 1️⃣ 注册新用户
curl -X POST http://localhost:3000/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","email":"alice@test.com","password":"123456"}'

# 2️⃣ 登录获取 Token
curl -X POST http://localhost:3000/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"alice@test.com","password":"123456"}'

# 3️⃣ 查看当前用户信息（需 Token）
curl http://localhost:3000/auth/profile \
  -H 'Authorization: Bearer <你的Token>'

# 4️⃣ 创建文章（需登录）
curl -X POST http://localhost:3000/posts \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <Token>' \
  -d '{"title":"NestJS教程","content":"内容","authorId":1,"tagIds":[1,2]}'

# 5️⃣ 文章列表（公开）
curl http://localhost:3000/posts

# 6️⃣ 创建标签（需 admin 角色）
curl -X POST http://localhost:3000/tags \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer <admin的Token>' \
  -d '{"name":"TypeScript"}'

# 7️⃣ 获取所有标签（公开）
curl http://localhost:3000/tags

# 8️⃣ 上传文件
curl -X POST http://localhost:3000/files/upload \
  -F "file=@/path/to/your-file.txt"

# 9️⃣ 动态模块演示
curl http://localhost:3000/dynamic-config

# 🔟 用户 CRUD（公开）
curl http://localhost:3000/users
curl http://localhost:3000/users/1
curl -X PATCH http://localhost:3000/users/1 \
  -H 'Content-Type: application/json' \
  -d '{"username":"new-name"}'
```

# 查看数据库数据
